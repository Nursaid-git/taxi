# Домен `places` — адреса пользователя

Избранные и недавние адреса клиента (чипы «Дом», «Работа» и список недавних на
главном экране).

## Сущности

| Таблица               | Зачем                                                  |
|-----------------------|--------------------------------------------------------|
| `public.saved_places` | Сохранённые адреса: Дом, Работа, произвольные.         |
| `public.recent_places`| Недавние адреса (история выбора) для быстрого повтора. |

## Решение: тут можно RLS-CRUD, а не функции

В отличие от `rides`/`drivers`, это **простые личные данные без переходов
состояния и без привилегий**. Поэтому здесь правильно дать пользователю полный
CRUD по своим строкам **через RLS** (`user_id = auth.uid()`), а не городить функции.

Это важный принцип: функции нужны там, где есть машина состояний (заказ) или
доверенная операция (модерация, начисление). Для «своих заметок» хватает RLS.

## Детали
- «Дом» и «Работа» — особые слоты: частичный уникальный индекс не даёт завести
  два «Дома» (`kind in (home, work)` — по одному на пользователя). Произвольных
  (`other`) может быть сколько угодно.
- Недавние дедуплицируются по адресу (`unique (user_id, address)`); повторный
  выбор обновляет `last_used_at` (через upsert из приложения).

## Состав
- `tables/00_enums.sql` — `place_kind`.
- `tables/01_saved_places.sql`, `02_recent_places.sql`.
- `indexes/01_places_indexes.sql`.
- `rls/01_places_rls.sql`.

> Применять после `auth`.
